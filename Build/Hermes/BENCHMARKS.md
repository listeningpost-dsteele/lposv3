# Objective

Verify that Strategic Planner completes realistic domain work accurately, within authority,
and using the LPOS output contract.

## Scenario

Provide a realistic request containing an objective, constraints, relevant context,
imperfect information, and one material tradeoff.

## Required behavior

- Stay inside the specialist's domain.
- State material assumptions.
- Identify risks and tradeoffs.
- Provide a recommendation and confidence.
- Define evidence that would validate the result.
- Escalate only when blocked.

## Failure criteria

Unsupported claims, hidden assumptions, excessive scope, vendor coupling, missing
confidence, missing evidence plan, or unnecessary questions.
