# 4.0.0

The baseline: LPOS v4.0.0 is the first single, integrated agent operating-system distribution — the operating specification, deterministic control plane, capability registry, executable schemas, Standing Operation workflows, adapters, database migrations, tests, installer, and CLI shipped together under one version.

## Release identity

- Product: LPOS, version 4.0.0
- Distribution: integrated, complete source-and-offline-install bundle
- Python requirement: 3.11 or later
- State backend: SQLite, with append-only audit events
- Default consequential-action mode: **record-only**

## What shipped

- The packaged LPOS v4 operating specification with the always-loaded Chip kernel and lazy component retrieval
- One-Principal authority and explicit materiality policy
- Typed runtime envelopes: TaskEnvelope, InterpretationContract, ArtifactSpecification, ReviewEnvelope, ActionPlan, ApprovalGrant, EvidenceRecord, DecisionRecord, CompletionReport
- Deterministic task and action state machines
- Capability-first routing across all 32 specialists by canonical ID
- Five-gate material-artifact review with a fresh isolated review context
- Exact-action approval bound to the canonical SHA-256 action payload, with verified Principal identities, expiry, replay protection, and one-time consumption
- Transactional SQLite state, append-only events, optimistic concurrency, checksummed migrations, idempotency, and operation leases
- 21 machine-readable Standing Operation workflows (SO-001 through SO-021)
- 53 fixed benchmark fixtures — one per specialist and one per Standing Operation
- 17 synchronized executable JSON Schemas
- A provider-neutral JSON subprocess protocol for model hosts
- A record-only consequential-action adapter and sandboxed local-file adapter for safe installation tests
- Cross-platform offline installation and a no-side-effect end-to-end verification flow

## Verification at release

From the release test report: the full source suite passed (128 tests, 310 subtests), all 17 JSON Schemas validated, all 53 deterministic core evaluations passed, and the bundled wheel was installed offline in a clean virtual environment where `pip check`, `lpos version`, `doctor`, a fresh database initialization, and the record-only demo all passed — repeated again from the extracted release ZIP.
