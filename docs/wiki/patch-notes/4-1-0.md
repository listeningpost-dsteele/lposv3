# 4.1.0

A feature release: LPOS gains its three user-facing modules, its living user guide, and three new Standing Operations to keep releases, connectors, and documentation honest.

## What is new

### Skill Evolution

LPOS can now improve its own skills from evidence behind a held-out validation gate. The engine ships as `lpos_engine.evolution`; the user-facing procedure ships as the `skill-evolution` skill under `spec/skills/skill-evolution`, alongside the existing packaged skills. The capability is offline (no model or network calls), staging-only (it cannot write a live skill by construction), and governed (independent review plus Principal approval for material skill changes, with before/after scores in the evidence ledger). Derived from Microsoft SkillOpt (MIT) and reimplemented independently; the lineage is recorded in `NOTICE-SKILLOPT.md`. See [Skill Evolution](/includes/skill-evolution.html).

### Hermes Project Dashboard

Every user now gets a single pane of glass for what their agents are doing: four buckets (Active, Research, Snoozed, Archive), snooze with presets and custom wake times, a browsable archive, and, the headline, every project and deliverable with its disk path viewable, copyable, and openable in one click, plus global search across project and file names. Local server at port 7373 (`lpos dashboard`), installed and launched by onboarding, auto-started with the system, state under `~/.hermes/dashboard/`. Existing users upgrading: the dashboard populates from your current Hermes directories with everything defaulting to Active. See [Hermes Project Dashboard](/includes/dashboard.html).

### Connector Health Monitor

An hourly audit of everything LPOS runs on, email, GitHub, cloud providers, MCP connectors, and self-built services, with auto-discovery, real authenticated checks, timeout-and-retry before anything is declared offline, and email alerting: one email per outage transition, one all-clear on recovery, no spam in between, a daily reminder after 24 hours, and a fallback channel if the email connector is itself down. On-demand via `lpos monitor audit`; state and the dashboard-readable `status.json` under `~/.hermes/monitor/`. See [Connector Health Monitor](/includes/connector-health-monitor.html).

### The LPOS User Guide wiki

This guide. Sources live in the repository under `docs/wiki/`, the site is built by `tools/build_wiki.py` from those sources plus the real system (the reference pages for Standing Operations, specialists, and skills are generated from the packaged catalog, index, and skill files), and every page carries the release version badge. See [How this wiki works](/documentation/how-the-wiki-works.html).

### New Standing Operations

- **SO-022 Release Publication**, publishes a release through the standing delegation and its gates (verifier passes, new version with changelog entry, secrets scan clean, independent review passed, destinations match), then commits, pushes, and updates the Drive artifact in place.
- **SO-023 Connector Health Audit**, the Connector Health Monitor's hourly audit as a Standing Operation, with one run record and one evidence record per cycle like every other operation.
- **SO-024 Documentation Drift Audit**, weekly safety net for this guide: diffs the modules, operations, and connectors found in the repository against the wiki's pages and files a task for anything undocumented that dodged the docs gate.

## What you will notice

After upgrading: the dashboard opens with your projects in it, the first connector audit report arrives, `lpos list-workflows`/`doctor` reflect the new operations, and this guide is available with a 4.1.0 badge. No changes to the approval model, the state database schema contract, or the record-only default for external actions.
