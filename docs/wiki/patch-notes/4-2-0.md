# 4.2.0

> **Security audit update, July 22, 2026:** this page describes the shipped feature intent. The v4.2.0 implementation was later found to overstate SOC 2 readiness. Treat its compliance output as a readiness diagnostic only, not a SOC 2 compliance claim or Type 2 attestation. See [4.2.0 security audit and SOC 2 readiness remediation](/patch-notes/4-2-0-soc2-readiness-remediation.html).

A feature release: LPOS gains the SOC 2 Compliance Guild, compliance codified into the
operating system, run autonomously, with staged remediation and an evidence trail.

## What is new

- **GUILD-038, the SOC 2 Compliance Guild**: owns the codified control framework (AICPA
  2017 Trust Services Criteria, revised 2022 points of focus) and the autonomous
  compliance loop. See [SOC 2 Compliance Guild](/includes/soc2-compliance.html).
- **`lpos_engine.compliance`**: 21 machine-checkable readiness controls mapped across
  CC1–CC9 plus Availability, Confidentiality, and Processing Integrity; every result
  carries evidence citing the exact files inspected. The shipped v4.2.0 self-audit must
  not be used to claim Type 2 operating effectiveness.
- **SO-025, SOC 2 Compliance Audit (daily)**: inventory → audit → staged remediation →
  report. Fixes for failing controls are built as copies in the staging test environment
  (`~/.hermes/compliance/staging/`) so live work is never broken, and adoption into the
  main system happens only through exact-action Principal approval.
- **The compliance page**: `~/.hermes/compliance/report.html`, problems, fixes, the
  audit log of changes, and project status, in one self-contained HTML page. The
  dashboard reads `compliance/status.json` alongside the monitor's health strip.
- **CLI**: `lpos compliance audit | report | status`.

## What you will notice

A new daily Standing Operation in `lpos list-workflows` (25 total), the compliance page
appearing under your Hermes root after the first run, and, if the audit finds gaps ,
staged fixes waiting for your approval rather than surprise changes to live files.
