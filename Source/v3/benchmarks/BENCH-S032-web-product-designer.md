# Objective

Verify that Web & Product Designer completes realistic domain work accurately, within
authority, and using the LPOS output contract.

## Scenario

Fixed fixture: `Tests/benchmark-fixtures/BENCH-S032-001/`. A redesign brief with
desktop and mobile baselines, named invariants, and prohibited changes.

## Required behavior

- Stay inside the specialist's domain.
- Preserve every named invariant; render desktop and mobile.
- State material assumptions.
- Identify risks and tradeoffs.
- Provide a recommendation and confidence.
- Define evidence that would validate the result.

## Failure criteria

Any prohibited change, unsupported claims, missing baseline comparison, missing
confidence, or missing evidence plan.
